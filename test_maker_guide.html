🏭 일일 테스트 제작 가이드 (V13.1)

👇 시험지 공장 가동하기 👇

🚀 일일 테스트 생성기 실행

Step 1. 기본 준비

  • 작업 폴더: 바탕화면의 kim-korean-homework 폴더
  • 필수 프로그램: GitHub Desktop (배포용)
  • PDF 파일: 시험지 파일이 있다면 미리 폴더에 넣어두세요. (예: daily_0114.pdf)

Step 2. 생성기 입력 (중요!)

1. 정보 입력:

  • 시험 제목: 엑셀 1행에 적을 제목과 똑같이 적으세요. (예: 1월 14일 어휘 테스트)
  • 시험지 링크: 같은 폴더에 있는 파일명이면 파일명.pdf만 입력.
  • 정답: 12345 처럼 숫자만 쭉 입력.

💡 학원 선택은 어디 갔나요?
V13.1부터는 선생님이 학원을 선택하지 않습니다.
학생이 시험 칠 때 직접 입력하게 바뀌었으니, 선생님은 문제만 내시면 됩니다! 👍

2. 파일 생성: [1. 코드 생성] 누르고 [2. 파일로 저장] 클릭.

3. 이동: 다운로드된 HTML 파일을 작업 폴더로 옮깁니다.

Step 3. 엑셀(상황판) 세팅

이 과정을 안 하면 점수가 안 들어옵니다! 🚨

  1. 구글 스프레드시트(숙제 관리표)를 켭니다.
  2. 시트1 (데이터 창고) 말고, 시트2 (종합상황판)으로 갑니다.
  3. 1행(제목줄)의 빈칸에 방금 만든 [시험 제목]을 적습니다.
  4. 이전 테스트의 점수 칸 수식을 복사해서, 새 테스트 칸으로 드래그합니다.

Step 4. 배포 및 전송

1. GitHub Desktop:

  • 새 파일이 + 초록색으로 떴는지 확인.
  • Summary 작성 후 [Commit to main] → [Push origin] 클릭.

2. 링크 만들기:

https://kim-korean.netlify.app/파일명.html

3. 학생 공지:
"얘들아 링크 보낸다. 학원 이름(러셀분당/시대인재 등) 정확하게 입력하고 풀어라!"

💡 문제 해결 (Troubleshooting)

Q. 학생들이 제출했는데 엑셀에 안 떠요.
A. 엑셀 1행에 적은 [제목]과 생성기에서 만든 [제목]이 토씨 하나라도 다르면 안 뜹니다. 띄어쓰기까지 확인하세요!

Q. 점수가 0점이고 흰색 줄이 생겨요.
A. 학생이 학원 이름 뒤에 공백(스페이스바)을 넣었을 확률이 높습니다. 엑셀 '시트1'에서 공백을 지워주세요.
